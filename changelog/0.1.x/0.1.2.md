# 0.1.2 — 2026-06-12

Dependency and framework refresh. No tool-surface changes.

## Added

- **Explicit identity** — `createApp()` now sets `name` and `title` to `nonprofit-explorer-mcp-server`, pinning the machine name on every client-facing surface instead of falling back to the scoped npm name.
- **Dockerfile `HEALTHCHECK`** — polls `/healthz` (interval 30s, timeout 5s, 3 retries) so orchestrators see container health.
- **`org.opencontainers.image.version`** label, stamped from a new `APP_VERSION` build arg.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into `bundle`; runs `mcpb clean` then strips dependency-shipped agent docs (`node_modules/**` `skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **`lint:packaging` checks 5–9** — bundle-content guards (dev dirs excluded at root; `.mcpbignore` patterns anchored so they don't strip nested runtime paths; critical runtime paths preserved; built `.mcpb` carries no `node_modules` agent docs) and an identity check that `name`/`title` in `createApp()` and manifest `display_name` equal the unscoped package name.
- **`check-framework-antipatterns` rule 4** — flags `z.coerce.boolean()` on env flags (`Boolean("false")` is `true`); recommends `z.stringbool()`. Comment lines are now skipped so a documented mention isn't a finding.

## Changed

- **`.mcpbignore`** — root dev-dir and config patterns anchored with a leading `/` (`skills/` → `/skills/`, etc.) so they match only at bundle root.
- **README client config** — example `mcpServers` keys renamed `nonprofit-explorer` → `nonprofit-explorer-mcp-server` across the bunx, npx, and Docker snippets.
- **Vendored skills** re-synced to the installed framework; adds the `orchestrations` and `techniques` skills.
