# 0.1.4 — 2026-06-20

Framework and tooling refresh. No tool-surface or behavior changes.

## Added

- **`Dependency Specifiers` devcheck step** — `scripts/check-dependency-specifiers.ts` hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json` dependency sections and the `bun.lock` `workspaces` map, catching the case where a `bun update --latest` writes a literal dist-tag into the lock and lets a later `bun install` re-resolve past the manifest range. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **`packaging.pluginManifests` config flag** (`devcheck.config.json`, default on) — enables `lint:packaging` check 10, validating `.claude-plugin/plugin.json` and `.codex-plugin/{plugin,mcp}.json` for non-empty descriptions, unscoped display names, and a full-name `npx -y` install arg. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))

## Changed

- **Vendored skills re-synced** to mcp-ts-core `^0.10.9` — `api-context` documents the new `ctx.content` media-block collector ([cyanheads/mcp-ts-core#239](https://github.com/cyanheads/mcp-ts-core/issues/239)); `api-config`, `api-telemetry`, and `polish-docs-meta` correct the `OTEL_SERVICE_NAME` seeding precedence ([cyanheads/mcp-ts-core#233](https://github.com/cyanheads/mcp-ts-core/issues/233)); `git-wrapup` and `orchestrations` carry the v1.4 tag-annotation and isolation guidance.

## Fixed

- **Fresh-scaffold devcheck guards** — `build-changelog.ts` (`--check`), `devcheck.ts`, and `check-framework-antipatterns.ts` now skip git-dependent checks cleanly when run before `git init`; `check-skill-versions.ts` skips a `SKILL.md` deleted from the worktree instead of crashing on the stale path. ([cyanheads/mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237), [cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [cyanheads/mcp-ts-core#243](https://github.com/cyanheads/mcp-ts-core/issues/243))
